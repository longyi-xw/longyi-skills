# 笔记总结模板

三种粒度各一套模板。**生成时把最终笔记整体放进一个 Markdown 代码块**，方便用户一次性复制粘贴到 Notion。所有语法都是 Notion 兼容的标准 Markdown：`#` 标题、`-` 列表、`1.` 有序列表、`>` 引用、表格、`- [ ]` 待办、行内 `代码`、`**加粗**`。

按需删减分区——空的分区（比如没有存疑项）直接删掉，不要留占位。

---

## 一、小节总结模板

适用于单个小节 / 单个概念。

```markdown
# [小节标题]

> 来源：《书名》· 第 X 章 · 第 Y 节 ／ 学习主题

## 一句话概括
（这一节到底在讲什么，用一句话说清）

## 核心概念
- **概念 A**：定义 / 解释
- **概念 B**：定义 / 解释

## 关键要点
1. 要点一
2. 要点二
3. 要点三

## 我的理解
（提炼并保留用户原笔记中有价值的表达 / 类比 / 例子）

## 待核对 · 存疑
- ⚠️ 与资料对不上、需进一步确认的点
- ❓ 暂时没弄清的问题

## 关键词
`关键词1` `关键词2` `关键词3`
```

**填写示例（节选）：**

```markdown
# 防抖 Debounce

> 来源：《JavaScript 设计模式与开发实践》· 高阶函数应用

## 一句话概括
让一个频繁触发的函数，在「停止触发后等待一段时间」才真正执行，避免无意义的高频调用。

## 核心概念
- **防抖 debounce**：n 毫秒内重复触发只算最后一次；常用于输入联想、窗口 resize。
- **闭包计时器**：用闭包保存 timer 引用，新触发到来时先 clear 旧 timer。

## 关键要点
1. 每次触发都重置计时器。
2. 与节流 throttle 的区别：防抖「等安静」，节流「按固定频率」。

## 我的理解
就像电梯门——只要有人进来就重新等几秒，没人了才关门。

## 关键词
`debounce` `闭包` `高阶函数`
```

## 二、章节总结模板

适用于一整章，聚合多个小节。需要各小节笔记作为素材。

```markdown
# 第 X 章 · [章标题] 总结

> 《书名》· 第 X 章 ｜ 涵盖小节：第 1 节…、第 2 节…

## 本章主线
（用一段话讲清这一章想解决什么问题、沿什么逻辑展开）

## 知识结构
| 小节 | 核心内容 | 关键结论 |
| --- | --- | --- |
| 第 1 节 … | … | … |
| 第 2 节 … | … | … |

## 核心概念清单
- **概念**：解释
- **概念**：解释

## 重点与难点
1. 重点：…
2. 难点：…（为什么难、怎么理解）

## 与前后章的联系
- 承接上一章：…
- 引出下一章：…

## 待核对 · 存疑
- ⚠️ …
- ❓ …

## 一句话自测
（合上书，用一句话复述本章——检验是否真的懂了）
```

## 三、全书总结模板

适用于整本书复盘。需要各章笔记作为素材。

```markdown
# 《书名》全书总结

> 作者 ｜ 阅读时间 ｜ 类别 / 主题

## 这本书在解决什么问题
（一段话：作者的核心关切与写作动机）

## 核心论点
1. 主张一
2. 主张二
3. 主张三

## 全书结构地图
| 章 | 主题 | 我的一句话 |
| --- | --- | --- |
| 第 1 章 | … | … |
| 第 2 章 | … | … |

## 关键概念词典
- **概念**：定义
- **概念**：定义

## 可迁移的方法 · 行动清单
- [ ] 我能实际用起来的做法 / 习惯
- [ ] …

## 我的批判性思考
- 认同：…
- 存疑 / 反例：…
- 与我已有经验的连接：…

## 一句话推荐语 & 关键词
推荐语：…
关键词：`关键词1` `关键词2` `关键词3`
```

## 通用注意事项

- **主干来自用户笔记**：模板里的内容主要来自用户的笔记和已确认的答疑结论，检索资料仅用于印证与补漏。
- **存疑显式标注**：未核实的内容统一放进「待核对 · 存疑」区，用 ⚠️ / ❓ 标注，不要写成定论。
- **不堆砌**：分区按需取舍，宁缺毋滥，保持清爽好读。
- **Notion 兼容**：避免使用 toggle、callout 等非标准 Markdown 语法；表格、待办、引用、行内代码均可被 Notion 正确识别。
